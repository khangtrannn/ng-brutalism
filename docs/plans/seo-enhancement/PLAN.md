Post-v0.1.0 release SEO work for the docs site at `https://ngbrutalism.khangtran.dev`
and the npm package `@ng-brutalism/ui`. Output of the `/grill-me` session of
2026-05-21.

This plan implements decisions reached across seven grilling questions. Each
section names the question that authored it so future readers can trace
"why this and not the alternative."

> GitHub + docs surfaces lives in [`audit-2026-05-22.md`](./audit-2026-05-22.md)

## 2. Decisions reached

### 2.1 Homepage split (Q2)

`/` is currently identical to `/docs/introduction/` and canonicalizes away from
itself. **Decision: split.** `/` becomes a dedicated marketing landing page;
`/docs/introduction/` stays as docs onboarding. Each self-references canonical.

### 2.2 Head-term target for `/` (Q3)

**Primary: "neobrutalism angular"** (capturing developers who already know the
visual style and are looking for Angular-native primitives). **Secondary
cluster: "neo-brutalist angular ui library", "brutalist angular components".**
Branded "ng brutalism" comes free.

Explicitly **not** targeting the broad "angular ui library" head term — too
generic and crowded for this stage.

### 2.3 Resolve `/docs/<component>/` duplicates (Q4)

Every component currently has two URLs serving byte-identical body content
(`/docs/button.page.ts` is an 11-line re-export of `/components/button.page.ts`).
**Decision: delete the `/docs/<component>/` wrapper routes entirely.** Sitemap
initially shrank to 19 URLs: `/`, two `/docs/*` pages, 15 `/components/*`
pages, and the portfolio showcase. A later launch-week content-depth exception
added `/docs/faq/`, bringing the generated route set to 20 URLs.

### 2.4 Per-component keyword pattern + H1 treatment (Q5)

For each of the 15 surviving `/components/<x>/` pages:

- **H1 treatment: hybrid.** Replace the existing `<p>Components</p>` eyebrow
  (e.g., `button.page.ts:22`) with a keyword-rich line like *"Neo-Brutalist
  Angular Component"*. H1 itself stays minimal ("Button").
- **Title tag pattern:** *"Neo-Brutalist Angular Button | Ng Brutalism"*
  (replaces *"Button | Ng Brutalism"*).
- **First paragraph rewrite:** lead with *"The neo-brutalist Angular Button
  component."* then describe behavior.
- **Dual-target:** `/components/dialog/` also targets *"angular brutalist modal"*
  (more common search term than "dialog"). `/components/marquee/` has no
  meaningful search volume — target brand cluster only.

### 2.5 JSON-LD restructure for entity authority (Q6)

- **`SoftwareApplication` blob: move from `apps/docs/index.html` to `/` only.**
  Currently emitted on every page; the lib is one entity with one home.
- **Enrich with:** `sameAs` (npm + GitHub repo), `softwareVersion` (synced from
  `libs/ui/package.json`), `license`, `codeRepository`, `downloadUrl`,
  `keywords`, `alternateName`, author `Person.sameAs` (GitHub profile).
- **`BreadcrumbList` JSON-LD** on every non-home page (2-level: Home > Button).
- **`TechArticle` JSON-LD** on each docs/component page (`headline`,
  `description`, `datePublished`, `dateModified`, `author`, `mainEntityOfPage`).

`sameAs` initial set: `https://www.npmjs.com/package/@ng-brutalism/ui`,
`https://github.com/khangtrannn/ng-brutalism`, `https://github.com/khangtrannn`.
Add Twitter/X, LinkedIn, personal site later via one-constant edit if desired.

### 2.6 Mechanical hygiene + `llms.txt` + external signals (Q7)

- **Sitemap generation moves to build time.** Reads route table, derives
  `<lastmod>` from `git log -1 --format=%cI` per page. Adds `/`, removes the
  16 `/docs/<component>/` entries.
- **`/llms.txt`** generated by the same build script. Reuses `PAGE_DESCRIPTIONS`
  from `docs-title-strategy.ts:16-90` — no new content writing. Skip
  `llms-full.txt`.
- **Meta tag hygiene in `apps/docs/index.html`:**
  - Remove `<meta name="keywords">` (identical on every page, ignored by Google,
    weakly negative signal).
  - Move `<meta name="google-site-verification">` to `/` only.
  - Add `<meta name="author" content="Khang Tran">`.
  - Add `<meta name="theme-color">` (brand yellow).
- **`og:type` per route:** `website` on `/`, `article` on every docs/component
  page. One-line change in `docs-title-strategy.ts`.
- **GitHub repo settings (manual, 5 min):** add topic tags `angular`,
  `angular-ui`, `angular-components`, `ui-library`, `component-library`,
  `neo-brutalism`, `neobrutalism`, `brutalist-design`, `tailwindcss`, `signals`,
  `zoneless`, `design-system`. Verify description + homepage URL.
- **Image alt audit** — verify every image has a meaningful alt (added to scope).
- **Core Web Vitals** — out of scope here, deferred to a separate ticket
  (needs real-device Lighthouse).

## 4. Execution order

Phased so each phase is independently shippable and the highest-impact /
lowest-blast-radius work lands first.

### Phase 1 — Structural fixes (highest SEO impact, smallest code change)

Goal: stop bleeding signal from duplicates and canonicalization, before adding
new content.

1. **Delete `/docs/<component>/` wrapper routes** (decision 2.3)
   - Delete 16 files in `apps/docs/src/app/pages/docs/` (every file except
     `installation.page.ts` and `introduction.page.ts`).
   - Remove their entries from sidebar nav in
     `apps/docs/src/app/docs/docs.navigation.ts`.
   - Remove their entries from `PAGE_DESCRIPTIONS` in
     `apps/docs/src/app/docs/docs-title-strategy.ts:55-85`.
   - Remove 16 lines from `apps/docs/public/sitemap.xml` (or just regenerate in
     Phase 4).
   - Verify dev server: `pnpm nx serve docs`, click through every component
     in the sidebar. No `/docs/<component>/` URL should remain reachable.

2. **Per-component title + eyebrow + first-paragraph rewrite** (decision 2.4)
   - Update `getDocsPageTitle()` in `docs-title-strategy.ts` so component
     routes map to *"Neo-Brutalist Angular {Component}"* instead of just
     *"{Component}"*. Non-component routes unchanged.
   - In each `apps/docs/src/app/pages/components/<x>.page.ts`, replace the
     `<p>Components</p>` eyebrow with *"Neo-Brutalist Angular Component"*
     (or *"Neo-Brutalist Angular Button"* per-page if we want each unique —
     decide during implementation, lean per-page).
   - Rewrite the first descriptive paragraph to lead with *"The neo-brutalist
     Angular {Component} component."* (the existing meta description from
     `PAGE_DESCRIPTIONS` is a usable template).
   - Dialog page: include "modal" once naturally.

### Phase 2 — Landing page (`/`)

Goal: own the head-term cluster + give entity-authority signals a home.

3. **Build `/` as a dedicated marketing page** (decisions 2.1, 2.2)
   - New file: `apps/docs/src/app/pages/(home).page.ts` (or wherever AnalogJS
     routes `/`). Today `/` resolves to `/docs/introduction/`'s prerendered
     HTML — break that link.
   - H1: *"The neo-brutalism UI library for Angular"* (covers neobrutalism +
     angular + ui library in one line).
   - First paragraph: contain the exact phrase *"neo-brutalist Angular UI
     library"* once and *"brutalist Angular components"* once, naturally.
   - Title tag: *"Ng Brutalism — Neo-Brutalist Angular UI Library"*.
   - Sections (suggested): hero + value props, install snippet, link to
     `/components/` (or directly to button if no index), link to showcase,
     link to GitHub + npm.
   - Reference the portfolio showcase site `https://ronit.io/` for landing-page
     pattern (per CLAUDE.md §5 — fetch the source before designing visuals).
   - Register the route in the title strategy with its own description.

### Phase 3 — Structured data

Goal: entity authority (D) + content-type signals (A).

4. **JSON-LD restructure** (decisions 2.5)
   - Remove the existing `SoftwareApplication` block from
     `apps/docs/index.html:36-57`.
   - On the new `/` page, emit an enriched `SoftwareApplication` (full
     `sameAs`, `softwareVersion`, `license`, `codeRepository`, `downloadUrl`,
     `keywords`, `alternateName`, `author.sameAs`). Version field sourced from
     `libs/ui/package.json` — read at build time via a small generator or
     inlined constant kept in sync by `scripts/sync-repository-metadata.mjs`
     (already exists per `package.json`).
   - Extend `DocsTitleStrategy.updateTitle()` to also inject:
     - A `BreadcrumbList` JSON-LD `<script>` per non-home page (Home > {Page}).
     - A `TechArticle` JSON-LD `<script>` per docs/component page.
   - Date fields: `datePublished` and `dateModified` from a manifest built at
     SSG time (`git log -1 --format=%cI <route-file>`).

### Phase 4 — Build-time generators

Goal: kill rot in sitemap; ship `llms.txt` for goal C.

5. **`apps/docs/scripts/build-seo-artifacts.mjs`** (decisions 2.6, partial)
   - Reads the route table (importable from the Analog router config or a
     small `routes.manifest.ts` exporting `[path, title, description, type]`).
   - Generates `apps/docs/public/sitemap.xml` with `<lastmod>` from
     `git log -1 --format=%cI` per route's page file. Includes `/`. Excludes
     `/docs/<component>/`.
   - Generates `apps/docs/public/llms.txt` reusing `PAGE_DESCRIPTIONS`.
   - Wire as a postbuild step in the docs project's `project.json`.

### Phase 5 — Meta tag hygiene

Goal: small, clean wins. Cheap to land last.

6. **`apps/docs/index.html` cleanup** (decisions 2.6)
   - Delete `<meta name="keywords">`.
   - Delete `<meta name="google-site-verification">` (move to `/` page head
     only — emitted via the title strategy on the home route).
   - Add `<meta name="author" content="Khang Tran">`.
   - Add `<meta name="theme-color" content="#FFD400">` (confirm exact brand
     yellow during implementation — match `--nb-yellow` token).

7. **`og:type` per route in `DocsTitleStrategy`** — `website` on `/`,
   `article` elsewhere. Single ternary in `updateTitle()`.

### Phase 6 — External + verification

Goal: off-site signals + sanity check.

8. **GitHub repo settings (manual)** — topic tags + description + homepage URL
   (decision 2.6).
9. **Image alt audit** — grep every `<img>` and `![](`) in
   `apps/docs/src/**/*.{ts,html}` and `libs/ui/README.md`; confirm every one
   has a meaningful alt. Fix any blanks.
10. **Post-deploy verification** —
    - View page source on 5 routes (`/`, `/docs/introduction/`,
      `/docs/installation/`, `/components/button/`, `/showcase/portfolio/`):
      confirm unique title, unique description, correct canonical, correct
      `og:type`, presence of `BreadcrumbList` (non-home) and `TechArticle`
      (docs pages).
    - `curl -sI https://ngbrutalism.khangtran.dev/docs/button/` should 404 (or
      ideally 301 to `/components/button/` if we add a redirect step — TBD).
    - Submit updated sitemap to Google Search Console.
    - Validate `SoftwareApplication`, `BreadcrumbList`, `TechArticle` JSON-LD
      via [Schema Markup Validator](https://validator.schema.org/) and
      [Rich Results Test](https://search.google.com/test/rich-results).

## 6. Success criteria

This plan is *done* when:

- `curl -sL https://ngbrutalism.khangtran.dev/ | grep -oP '<link rel="canonical" href="\K[^"]+'`
  returns `https://ngbrutalism.khangtran.dev/` (self-referential — Phase 2 lands).
- `curl -sI https://ngbrutalism.khangtran.dev/docs/button/` returns 404 or 301
  (Phase 1 lands).
- Every component page's `<title>` contains "Neo-Brutalist Angular" (Phase 1).
- `https://ngbrutalism.khangtran.dev/llms.txt` returns markdown with 19 URL
  entries, including `/showcase/portfolio/` (Phase 4).
- Sitemap contains `<lastmod>` on every URL and no `/docs/<component>/` entries
  (Phase 4).
- Schema validator returns 0 errors on `/`, `/docs/introduction/`,
  `/components/button/` (Phase 6).
- GitHub repo page shows topic tags matching the keyword cluster (Phase 6).

This plan is *working* (separate from done) when, over 60–90 days post-deploy:

- Google Search Console shows impressions for queries containing
  "neobrutalism angular", "brutalist angular", "ng brutalism".
- Branded query "ng brutalism" returns `/` as the first result (not
  `/docs/introduction/`).
- Search for `site:ngbrutalism.khangtran.dev` shows 19 indexed URLs, not 33.

  Recommend: accept 404, revisit if Search Console shows real traffic to those
  URLs.
- Twitter/X, LinkedIn, personal-site URLs for `sameAs` — fill in later via the
  constant in the title strategy or the new home page component.

  token.
- Component eyebrows use per-page text, e.g. `Neo-Brutalist Angular Button`.
- The generated route set is 20 URLs because the portfolio showcase and the
  later `/docs/faq/` content-depth page are retained alongside `/`, two original
  docs pages, and 15 component pages.
